## 1️⃣ مقدمة (Introduction)

### 🎯 الغرض من المستند
المستند ده بيوصف متطلبات منصة تعليمية متكاملة للطلاب في المراحل الدراسية المختلفة (ابتدائي - إعدادي - ثانوي). المنصة بتدي حلول شاملة للمدرسين والطلاب لإدارة العملية التعليمية.

### 🎭 نطاق المشروع
المنصة بتغطي:
- إدارة الهيكل الدراسي (مراحل - صفوف - مواد)
- إدارة الكورسات (أساسي - مراجعة - تكثيف لكل مادة)
- إدارة المحتوى التعليمي (وحدات - دروس - فيديوهات)
- نظام امتحانات متكامل مع AI
- نظام تقارير وتحليل أداء
- نظام اشتراكات ومدفوعات للكورسات

### 🎯 الأهداف الرئيسية
- توفير منصة سهلة الاستخدام للطلاب والمدرسين
- استخدام AI لتحسين تجربة التعلم
- إدارة شاملة للعملية التعليمية
- توفير تقارير دقيقة لتحليل الأداء

## 2️⃣ وصف عام للنظام (Overall Description)

### 🏗️ معمارية النظام
النظام هيبقى Web-based application معمول بـ:
- **Frontend**: Blades 
- **Backend**: Laravel 
- **Database**: mysql
- **AI Integration**: grok


### 👥 المستخدمين المستهدفين
1. **المدرس**: مسئول عن إدارة المحتوى والطلاب والامتحانات
2. **الطالب**: بستخدم المنصة للمذاكرة وحل الامتحانات

### 🔄 التفاعل مع الأنظمة التانية
- **Payment Gateway**: للمدفوعات والاشتراكات
- **Email Service**: للإشعارات
- **AI Services**: لتوليد وتصحيح الامتحانات

## 3️⃣ مميزات النظام (System Features)

### 🏫 إدارة الهيكل الدراسي
**الوصف**: المدرس يقدر يدير البنية التعليمية كاملة

**المميزات**:
- إضافة/تعديل/حذف مراحل دراسية (ابتدائي - إعدادي - ثانوي)
- إضافة/تعديل/حذف صفوف داخل كل مرحلة (أولى ابتدائي - ثانية ابتدائي...)
- إضافة/تعديل/حذف مواد لكل صف (رياضيات - علوم - لغة عربية...)

**مثال**: مدرس رياضيات يضيف مادة "الجبر" لصف "أولى ثانوي"

### � إدارة الكورسات
**الوصف**: كل مادة ممكن تحتوي على أكتر من كورس

**المميزات**:
- إنشاء كورسات مختلفة لنفس المادة (أساسي - مراجعة - تكثيف)
- تحديد سعر ومدة كل كورس
- ربط الكورسات بالمواد والصفوف
- إدارة اشتراكات الطلاب في الكورسات

**مثال**: مادة الرياضيات تحتوي على:
- كورس أساسي (100 جنيه/شهر)
- كورس مراجعة شاملة (150 جنيه/شهر)
- كورس تكثيف ليلة الامتحان (200 جنيه/أسبوع)

### �‍🎓 إدارة الطلاب
**الوصف**: إدارة بيانات الطلاب وربطهم بالصفوف

**المميزات**:
- تسجيل طالب جديد (اسم - بريد إلكتروني - كلمة مرور)
- ربط الطالب بصف معين
- عرض بيانات الطالب (درجات - امتحانات - اشتراكات)
- تعديل بيانات الطالب

**مثال**: إضافة طالب "أحمد محمد" في صف "ثالثة إعدادي" مادة "فيزياء"

### 📚 إدارة المحتوى التعليمي
**الوصف**: تنظيم وتقديم المحتوى التعليمي لكل كورس

**المميزات**:
- كل كورس فيه وحدات (Units)
- كل وحدة فيها دروس (Lessons)
- كل درس فيه:
  - فيديو تعليمي
  - ملف PDF للشرح
  - ملاحظات إضافية

**مثال**: كورس "الرياضيات - المراجعة الشاملة" يحتوي على وحدة "المعادلات" وفيها درس "المعادلات الخطية"

### 📝 نظام الامتحانات
**الوصف**: إنشاء وإدارة الامتحانات

**المميزات**:
- إنشاء امتحان يدوي (أسئلة اختيار من متعدد - أسئلة مقالية)
- استخدام AI لتوليد امتحان تلقائي
- تحديد الصف والمادة للامتحان
- إرسال الامتحان للطلاب
- تحديد وقت الامتحان

**مثال**: مدرس يولد امتحان "10 أسئلة اختيار من متعدد" لمادة "الكيمياء" صف "ثانية ثانوي"

### 🤖 مساعد AI
**الوصف**: استخدام الذكاء الاصطناعي لتحسين التعلم

**المميزات**:
- تصحيح الامتحانات تلقائي
- تحليل أداء الطالب وتحديد نقاط الضعف
- توليد أسئلة إضافية حسب مستوى الطالب
- الإجابة على أسئلة الطلاب

**مثال**: الطالب يسأل "اشرح لي نظرية فيثاغورث" وال AI بيجاوب بشرح مبسط

### 📊 التقارير والتحليل
**الوصف**: عرض تقارير مفصلة عن الأداء

**المميزات**:
- تقرير أداء الطالب (درجاته - عدد الامتحانات - نسبة النجاح)
- تقرير أداء المادة (متوسط درجات الطلاب - أصعب الأسئلة)
- تقرير المدفوعات (اشتراكات ناجحة - فاشلة)

**مثال**: عرض تقرير "أحمد محمد" بيظهر إنه حصل على 85% في امتحانات الرياضيات

### 💰 نظام المدفوعات
**الوصف**: إدارة اشتراكات الطلاب في الكورسات

**المميزات**:
- عرض أسعار الاشتراكات لكل كورس
- معالجة المدفوعات عبر Payment Gateway
- تسجيل عمليات الدفع (ناجحة - فاشلة)
- إشعارات حالة الدفع
- تجديد الاشتراكات التلقائي

**مثال**: طالب يدفع 150 جنيه للاشتراك في كورس "الفيزياء - المراجعة الشاملة" لمدة شهر

## 4️⃣ أدوار المستخدمين (User Roles)

## 5️⃣ المتطلبات الوظيفية (Functional Requirements)

### 📝 User Stories للمدرس

**UC1: إدارة مادة جديدة**
```
كد مدرس، 
أريد إضافة مادة جديدة، 
علشان أقدر أدير محتواها للطلاب.

السيناريو:
1. أدخل على قسم المواد
2. أضغط على "إضافة مادة جديدة"
3. أدخل اسم المادة "الرياضيات"
4. أختار الصف "أولى ثانوي"
5. أضغط "حفظ"
6. تظهر رسالة تمت الإضافة بنجاح
```

**UC2: إنشاء امتحان**
```
كد مدرس،
أريد إنشاء امتحان لمادة معينة،
علشان أقدر أختبر مستوى الطلاب.

السيناريو:
1. أدخل على قسم الامتحانات
2. أضغط على "إنشاء امتحان جديد"
3. أختار المادة "الفيزياء"
4. أختار الصف "ثانية ثانوي"
5. أختار "توليد بواسطة AI"
6. أدخل عدد الأسئلة "10"
7. أضغط "إنشاء"
8. يظهر الامتحان تم إنشاؤه
```

**UC3: عرض تقرير طالب**
```
كد مدرس،
أريد عرض تقرير أداء طالب معين،
علشان أعرف مستواه ونقاط ضعفه.

السيناريو:
1. أدخل على قسم التقارير
2. أختار "تقرير طالب"
3. أدخل اسم الطالب "أحمد محمد"
4. أضغط "عرض"
5. يظهر التقرير بدرجاته ونسبة نجاحه
```

## 6️⃣ المتطلبات غير الوظيفية (Non-Functional Requirements)

### ⚡ الأداء (Performance)
- **Response Time**: الصفحات تload في أقل من 3 ثواني
- **Concurrent Users**: النظام يدعم 1000 يوزر في نفس الوقت
- **Video Streaming**: فيديوهات HD بتعمل بدون lag

### 🔒 الأمان (Security)
- **Authentication**: تسجيل دخول آمن بـ JWT
- **Data Encryption**: البيانات الحساسة متشفرة
- **Payment Security**: مدفوعات آمنة بـ PCI Compliance
- **Data Backup**: نسخ احتياطي يومي للبيانات

### 📱 قابلية الاستخدام (Usability)
- **Responsive Design**: يشتغل على Mobile و Tablet و Desktop
- **Arabic Support**: دعم كامل للغة العربية RTL
- **Easy Navigation**: تصميم بسيط وسهل الاستخدام
- **Accessibility**: يتبع معايير WCAG 2.1

### 🛠️ الموثوقية (Reliability)
- **Uptime**: النظام متاح 99.9% من الوقت
- **Error Handling**: رسائل خطأ واضحة ومفيدة
- **Data Integrity**: ضمان عدم فقدان البيانات

### 📈 قابلية التوسع (Scalability)
- **Horizontal Scaling**: نقدر نضيف servers عند الحاجة
- **Database Scaling**: نقدر نوسع الـ database
- **CDN Integration**: فيديوهات وملفات من CDN

## 7️⃣ نظرة عامة على قاعدة البيانات (Database Overview)

### 📊 الجداول الرئيسية (Tables)

```sql
-- المستخدمين
Users (id, name, email, password, role, created_at, updated_at)

-- المراحل الدراسية
Stages (id, name, description, created_at)

-- الصوف
Grades (id, stage_id, name, description, created_at)

-- المواد
Subjects (id, grade_id, name, description, created_at)

-- الكورسات
Courses (id, subject_id, name, description, price, duration, type, created_at)

-- الوحدات
Units (id, course_id, name, description, order_index, created_at)

-- الدروس
Lessons (id, unit_id, title, video_url, pdf_url, notes, order_index, created_at)

-- الامتحانات
Exams (id, course_id, title, description, duration_minutes, ai_generated, created_at)

-- أسئلة الامتحانات
Questions (id, exam_id, question_text, question_type, options, correct_answer, points)

-- إجابات الطلاب
StudentAnswers (id, exam_id, student_id, question_id, answer_text, score)

-- اشتراكات الطلاب
Subscriptions (id, student_id, course_id, start_date, end_date, payment_status, amount)

-- المدفوعات
Payments (id, subscription_id, amount, payment_method, status, transaction_id, created_at)
```

### 🔗 العلاقات الرئيسية
- User 1:1 Student/Teacher
- Stage 1:N Grades
- Grade 1:N Subjects
- Subject 1:N Courses
- Course 1:N Units
- Unit 1:N Lessons
- Course 1:N Exams
- Exam 1:N Questions
- Student N:M Courses (عن طريق Subscriptions)

## 8️⃣ معمارية النظام (System Architecture)

### 🏗️ High-Level Architecture

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   Frontend      │    │    Backend      │    │   Database      │
│   (React.js)    │◄──►│   (Node.js)     │◄──►│  (PostgreSQL)   │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                       │                       │
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   CDN           │    │   AI Services   │    │   File Storage  │
│   (Videos/PDF)  │    │   (OpenAI)      │    │   (AWS S3)      │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### 🔄 Data Flow
1. **User Request**: Frontend يبعت API request للـ Backend
2. **Authentication**: Backend يتحقق من الـ JWT token
3. **Business Logic**: Backend ينفذ الـ logic المطلوب
4. **Database Operations**: يتعامل مع PostgreSQL
5. **AI Integration**: لو محتاج AI، بيتعامل مع OpenAI API
6. **Response**: Backend يرجع response للـ Frontend

### 🛡️ Security Layers
- **API Gateway**: لتأمين الـ API endpoints
- **Rate Limiting**: منع الـ abuse
- **Input Validation**: تأمين المدخلات
- **CORS**: تحكم في cross-origin requests

## 9️⃣ افتراضات وقيود (Assumptions and Constraints)

### 📋 الافتراضات (Assumptions)
- المستخدمين عندهم إنترنت جيد
- المدرسين عندهم خبرة في استخدام الكمبيوتر
- الطلاب عندهم أجهزة حديثة (Smartphones/Tablets/Laptops)
- المدفوعات هتتعامل معها Payment Gateway معتمدة في مصر

### ⚠️ القيود (Constraints)
- النظام هيبقى باللغة العربية بس
- الفيديوهات لازم تكون بصيغة MP4
- ملفات PDF لازم تكون أقل من 50MB
- الامتحانات الواحد ما يقدرش يدخله أكثر من مرة
- الاشتراكات لمدة شهر بس (في النسخة الأولى)
